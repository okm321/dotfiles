# Design Rules

UI デザインを実装する際のルール。

参照元: https://www.adhamdannaway.com/blog/ui-design/ui-design-tips-14

## スペーシング

- 要素間のスペースは**関連性の強さ**で決める。関連が強いほど近く、弱いほど離す
- 8pt グリッドシステム（8, 16, 24, 32, 48...）をベースにする
- 不要なコンテナ（カード、ボーダー）を増やす前に、スペーシング・類似性・アラインメントでグルーピングできないか検討する

## コントラスト

- **テキスト（18px 以下）**: 背景とのコントラスト比 4.5:1 以上
- **テキスト（18px 超）**: コントラスト比 3:1 以上
- **UI 要素**（ボタン、フォーム、インタラクティブ要素）: コントラスト比 3:1 以上。不足する場合はボーダーを追加する

## 色に依存しない

- 色だけで情報を伝えない。下線、アイコン、ボーダー、テキストラベルなど追加の視覚的手がかりを併用する
- 例: エラー表示は赤色 + アイコン + テキスト

## ボタン

- **プライマリボタンは画面に1つ**。複数あると注目が分散し混乱を招く
- タップターゲットは最低 48x48pt、ボタン間は最低 8pt の間隔
- よく使うボタンはさらに大きくする

## レイアウト・アラインメント

- **アラインメントの種類は最小限に**。基本は1種類（左揃え）に統一し、複数の揃え方を混在させない
- 一貫性を保つ: 角丸、スペーシングパターン、視覚的処理を画面全体で揃える

## タイポグラフィ（デザイン観点）

- 大きい見出しは**レタースペーシングを詰める**（視覚的に引き締まる）
- フォントウェイトは **Regular と Bold の2種類**に絞る。多すぎると散漫になる
- アイコンとテキストを並べる場合、ウェイト・色・サイズのバランスを取り視覚的な重みを揃える

## 情報の可視性

- 重要なアクションやコンテンツをメニューの中に隠さない。スペースがあるなら直接表示する
- ミニマリズムとシンプルさを混同しない。削りすぎ・隠しすぎはユーザビリティを損なう
- 判断に必要な情報は省略せず表示する
