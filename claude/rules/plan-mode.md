# Plan Mode Rules

planモード使用時のルール。

## Plan ファイル名

- 内容を表す簡潔な英語3〜4語をハイフン区切りにする
- 例: `add-auth-middleware.md`, `refactor-db-queries.md`

## Codex レビュー

planモードで計画を作成した後、ExitPlanMode を呼ぶ前に以下を実行する：

1. **Codex MCP を使って計画をレビュー**
   - `codex` MCPサーバーの `ask` ツールを使用
   - 以下の観点でレビューを依頼:
     - 計画の妥当性（実装方針やアーキテクチャの穴がないか）
     - セカンドオピニオン（別の視点からのフィードバック）

2. **レビュー結果の反映**
   - Codexからのフィードバックを計画に反映
   - 重要な指摘があれば計画を修正

3. **ユーザーへの報告**
   - Codexレビューの要約をユーザーに共有
   - 計画の修正点があれば説明

## レビュー依頼のプロンプト例

```
この実装計画をレビューしてください：

{計画の内容}

以下の観点でフィードバックをお願いします：
1. 実装方針に穴や見落としはないか
2. より良いアプローチはあるか
3. 潜在的なリスクや注意点
```
